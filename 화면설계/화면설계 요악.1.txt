<1>
화면 설계
- 사용자 요구사항 UI,UX 표준 및 지침, 소프트웨어 아키텍처를 바탕으로 사용자 인텊이스의 구조, 흐름, 구성 요소 등을 설계하는 작업

목적
- 1. 사용자 편의성 확보 2. 시스템 구현 전 미리 UI 흐름 확인, 오류 줄이기 3. 개발자와 디자이너, 클라이어트 간의 소통 도구로 활용

포함되는 것
- UI (흐름도, 구성요소 배치도), 프로토타입, 유스케이스

용어 정리
- UI / 사용자 인터페이스, 사용자가 보는 화면
- U / 사용자 경험, 사용자가 느끼는 편리함/ㅇ;해도
- 프로토타입 / UI를 시각적으로 보여주는 샘플 화면
- 흐름 설계 / 화면 간 이동 순서를 ㅏ타낸 다이어그램
- 상세 설계 / 각 화면에 어떤 기능과 정보가 있는지 구체적으로 표시

<2>
소프트웨어 아키텍처
- 소프트웨어를 구성하는 전체 구조와 원칙을 설계하는 작업
- 소프트웨어 개발 전 전체적 구조를 먼저 설계하는 것
- 예시 ) 어떤 화면이 어떤 서버와 연결될지

목적
- 복잡한 시스템을 체계적으로 구성하기 위해
- 기능 분리와 책임 분담을 명확히 하기 위해
 - 유지보수와 확장성을 고려한 설계를 위해
- 디자인/개발/운영팀 간의 소통 기준 제공

화면 설계와의 관계
- 화면 설계는 소프트웨어 아키텍처의 한 부분
- 아키텍처에서 정해준 흐름과 기준으로 바탕으로 화면을 설계

UI 설계와의 연계
- 아키텍처에서 제시된 지침에 따라 화면 흐름과 구성 요소 설계
- 품질속성(기능상,효율성,사용성 등)을 고려한 UI 요구사항 반영

활용 예시
- 아키텍처 설계 시 고려한 모듈 구조를 UI 네비게이션과 대응
- 기능 흐름도를 통해 요구기능 정렬

<3>
UI 요구사항 분석
- 사용자와 시스템이 상호작용하는 화면UI에 어떤 기능과 정보가 필요한지를 파악하고 정리하는 것
- 사용자가 어떤 걸 보고 파악하는지 사용자 입장에서 미리 예측하고 정리하는 작업

하는 일
- 1. 사용자의 목적 파악하기
	사용자가 왜 이 시스템을 쓰는지 어떤 작업을 하려는지를 조사
- 2. 필요한 기능 정리하기
	예>로그인,검색,목록보가,저장 등 어떤 동작이 필요한지 파악
- 3. 입력/출력 요소 확인하기
	어떤 입력창이 필요한지 어떤 결과가 화면에 보여야 하는지를 결정]]여기까지 중요+
- 4. 화면 흐름 예측하기
	버튼을 누르면 어떤 화면으로 넘어가는지를 설꼐 흐름으로 그려보는 것
- 5. UI 설계 원칙에 맞게 정리하기
	직관적이고 단순하고 예측 가능한 방식으로 구성해야 함

예시
- 도서 검색 화면 -> 도서 상세 정보 -> 대출 버튼 클릭 -> 대출 완료 화면

<4>
품질 요구사항 특성(ISO/IEC 9126 기준) -> 구)소프트웨어 국제 표준 기준. 2015로 기준 바꼈지만 여전히 중요한 기준으로 활용중
- 1. 기능성
시스템이 어떤 기능을 얼마나 정확하게 수행하는지를 평가하는 것
	1) 적절성 : 사용자의 목적에 맞는 기능을 제공하느 것
	2) 정밀성 : 정확한 결과를 제공하는 것
	3) 상호 운용성 : 다른 시스템과 연동이 잘 되는 것
	4) 보안성 : 데이터를 안전하게 보호하는 것
	5) 호환성 : 구격이나 표준을 잘 지키는 것
- 2. 신뢰성
오류 없이 안정적으로 작동하는지를 평가하는 것
	1) 성숙성 : 버그 없이 잘 작동하는 것
	2) 고장 허용성 : 오류가 나도 멈추지 않고 어느 정도 유지되는 것
	3) 회복성 : 오류가 나도 빨리 복구되는 것
- 3. 사용성
사용자가 쉽게 이해하고 사용할 수 있는지를 평가하는 것
	1) 이해성 : 기능과 흐름을 쉽게 이해할 수 있는 것
	2) 학습성 : 처음 쓰는 사람도 금방 배울 수 있는 것
	3) 운용성 : 원하는 작업을 쉽게 할 수 있는 것
-4. 효율성
적은 자원으로 빠르게 작동하는지를 평가하는 것
	1) 시간 효율성 : 반응 속도가 빠른 것
	2) 자원 효율성 : 메모리, CPU 등을 적게 사용하는 것

-5. 유지보수성
나중에 수정하거나 확장하기 쉬운지를 평가하는 것
	1) 분석성 : 문제가 생기면 원인을 빨리 찾을 수 있는 것
	2) 변경성 : 쉽게 수정할 수 있는 것

<6>
UI 레이아웃 설계 요소
- 화면 안에 로고, 메뉴, 콘텐츠, 버튼 같은 요소들을 어떤 구조로 배치할 지 결정하는 것.
- 사용자가 화면을 펴하게 보고 빠르게 이해할 수 있게 만드는 것

UI 레이아웃 설계의 주요 요소
- 1. TOP 영역(상단 영역)
- 2. Left 영역( 좌측 영역)
	화면 왼쪽에 위치해 보조 메뉴나 배너 보여주기
	화면 이동에 도움 되는 서브 메뉴 구조 제공
- 3. Contents 영역(콘텐츠 영역)
사용자가 직접 작업하거나 정보를 확인하는 중심 공간 배치
- 4. Footer 영역(하단 영역)
- 5. 페이지 폭 설정(반응형 대응)
사용자 화면 크기에 맞춰 자동으로 레이아웃이 조정되록 설계하는 것
- 6. 네비게이션 설계
사용자가 어디에 있는지 어디로 이동할 수 있는지를 쉽게 알 수 있게 설계하는 것
	메뉴 위치 일관성 유지
	현재 위치 강조 표시(예: 강조된 메뉴 색상)
	드롭다운, 탭, 사이드 메뉴
-> UI 레이아웃 설꼐는 화면을 어떤 구조로 나누고 각 영역에 어떤 요소를 배치할 지 정하는 것

네비게이션 구조와 모듈 뷰
- 사용자가 여러화면을 어떻게 오갈 수 있게 할지 그 이동 구조를 설계하는 것

네비게이션 구조 종류
유형 / 특징 / 사용하는 경우

네비게이션 설계 시 고려하는 것
- 메뉴 간 관계성 명확히 표현

모듈 뷰
- 시스템 구성 기능을 논리적 단위(모듈)로 나누고
	1) 분할 타입
	2) 사용 타입
	3. 일반화 타입
	4. 레이어 타입

모듈 뷰 설계 시 고려하는 것
- 각 모듈 기능과 책임 명확히 정리
- 응집고 높이고 결합도 낮게 설계

정리
- 네비게이션 구조는 사용자가 어떤 화면으로 어떻게 설계하는지 나타내는 것

UI 프로토타입 설계

목적
- 화면 흐름이 올바른지 검톤
- 사용자 요구사항이 제대로 반영됐는지 확인하기

	- 시스템이 회원카드를 요청함
	- 사용자가 카드 인식
	- 시스템이 도서 인식을 요청
	- 사용자가 도서르 인식
	- 시스템이 결과를 출력
